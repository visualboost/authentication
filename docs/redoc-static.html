Authentication and User Management API (1.0.0)

Download OpenAPI specification:Download

This API provides endpoints for managing user authentication, authorization, and profile operations. It includes features like user registration, login, two-factor authentication, password and email updates, and various system-level functionalities such as retrieving hooks and states. Designed to support secure and scalable authentication workflows.

System

System-related endpoints, such as XSRF token retrieval.

Get an XSRF token

Retrieve an XSRF token to avoid CSRF attacks. This should be your first API call.

Responses

http://localhost:80/api/system/xsfr

Get privacy policy settings

Returns whether the privacy policy is displayed and its URL if available.

Responses

http://localhost:80/api/system/privacypolicy

Response samples

  • 200
Content type
application/json
{
  • "showPrivacyPolicy": true,
  • "privacyPolicyUrl": "https://example.com/privacy"
}

Get system initialization state

Checks if the system has been initialized by verifying whether an admin user exists. Returns INITIALIZED if the admin user exists, or NOT_INITIALIZED otherwise.

Responses

http://localhost:80/api/system/state

Response samples

  • 200
Content type
application/json
{
  • "state": "INITIALIZED"
}

Get system hooks

Retrieve all hooks configured in the system. The response contains keys representing hook types and their corresponding URLs.

required
string

XSRF token obtained from the /api/system/xsfr endpoint

Responses

http://localhost:80/api/system/hooks

Authentication

password

Confirmation

Endpoints related to confirmation processes, such as registration confirmation and password reset confirmation.

Confirm registration

Triggered when the registration confirmation email button is clicked. The user will be redirected to a 'confirmed' view showing successful registration.

query Parameters
userId
required
string

The ID of the user being confirmed.

Responses

http://localhost:80/api/confirm/registration

Confirm password reset

Triggered if the reset password link in the 'Confirm reset password' email is clicked. Validates if a valid password reset modification object exists and redirects the user accordingly.

query Parameters
token
required
string

The token obtained after requesting a password reset.

Responses

http://localhost:80/api/confirm/password/reset

User

Get user details

Retrieves details of a user by their ID. Only admins or the user themselves can access this endpoint. The Bearer token required for authorization can be obtained by calling /api/authentication/signin.

path Parameters
userId
required
string

The ID of the user to retrieve.

header Parameters
Authorization
required
string

Bearer token for authentication, obtained from /api/authentication/signin.

x-xsrf-token
required
string

XSRF token obtained from the /api/system/xsfr endpoint

Responses

http://localhost:80/api/user/{userId}
